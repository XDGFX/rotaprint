Check Mode

Check mode is used to verify GCODE files are accepted by the printer,before actual printing occurs. By enabling check mode the GCODE will be streamed to the printer asnormal, but no movement will occur.

An output log will be presented once complete (will take as long as a normal print), forcheckingand verification. If you are happy with the output, disable check mode for real printing.

Manual Command

You can manually send commands to the backend for testing and debugging. It is recommended to keepthe logs window open while using this. Incorrect commands can break the backend and require a fullrestart - or worse!

Use at your own discretion.

Command Payload Description Response
EST NONE Emergency Stop "DONE"
BFH NONE Buffer hold; following messages are held
BFR NONE Buffer release; held messages are sent as one
LEN FLOAT Length of part to print on
BAT INT Number of parts in batch (1-5)
RAD FLOAT Radius of part to print on
CHK BOOL Enable check mode
DBS STRING Database setting update
GRB STRING Send manual code to GRBL
GCD NONE Following message is GCODE blob
PRN NONE Print now
FTS NONE Fetch all current settings Settings array
RQV STRING Request value of variable Current value
RCN NONE Reconnect to machine "DONE"
LOG NONE Request log messages Logs
